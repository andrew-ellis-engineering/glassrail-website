Reliability infrastructure for agentic workflows

Agentic workflows you can see through.

Glassrail turns a task into a validated DAG of typed nodes. Each node runs with fresh declared context, model calls route through deterministic tiers, and the run leaves events, traces, approvals, and evals behind.

02

Why plans, not loops

Opaque loops are hard to operate.

thought · tool · observe · thought · ?

What disappears in a loop

  • Which plan was accepted before tools ran.
  • Which context each step actually needed.
  • Whether tool use is reviewable before it happens.
  • When model routing changed cost or reliability.
  • How the final answer connects back to the work.
plan
validate
execute
result

What Glassrail makes explicit

  • The task becomes a typed, acyclic graph.
  • The graph is validated before execution.
  • Each node receives only declared upstream context.
  • Provider tiers fall through by deterministic rules.
  • Events, traces, and evals turn behavior into evidence.
03

Core tenets

The rails are concrete.

plan_ready

Plan as document

A run starts with an inspectable plan, not a hidden chain of model choices. The accepted graph can be displayed, reviewed, replayed, and compared. 

{"nodes":[{"id":1,"type":"tool"},{"id":2,"type":"synthesis"}]}
context_needed

Fresh context per node

Nodes only see the upstream outputs they declared. Context hygiene becomes an execution rule instead of a prompt wish.

cycle rejected

Validated before it runs

The validator enforces DAG invariants, topological order, nesting limits, and node-shape rules before any tool or model node executes.

tier_router

Deterministic routing

Providers are dumb. The router owns timeout and fallthrough, making model choice explicit enough to reason about cost and reliability.

  1. tier 0 · local
  2. tier 1 · cloud
  3. tier 2 · reasoning
  4. tier 3 · fallback
branch_decided

Branching is explicit

Decision nodes make control flow visible. Untaken branches are skipped, not erased, so the audit log still explains what did not run.

eval-gated

Releases earn their way out

Model-quality evals report pass@k for capability and pass^k for reliability. Remaining misses become named ratchets, not buried caveats.

04

The audit log replays

The final answer is not the only artifact.

Headless runs can emit a machine-readable envelope with the accepted plan, normalized trajectory, tiers used, branch decisions, status, and final output.

glassrail run --json "compare queues for retry-safe jobs" 
{
  "status": "completed",
  "plan": {"nodes": [{"id": 1, "type": "decision"}, {"id": 2, "type": "result"}]},
  "trajectory": [
    {"tool": "decision", "tier_used": 0, "branch_taken": "queue"},
    {"tool": "result", "tier_used": 1, "confidence": 0.84}
  ],
  "result": "Use a message queue for retry-safe background jobs..."
}
